
# Разбивка на подзадачи
### Бэкенд:
1. Создать Django проект и приложение
2. Настроить модели для хранения курсов валют
3. Реализовать API endpoint /get-current-usd/
4. Интегрировать внешнее API для получения курса
5. Реализовать механизм кэширования/паузы 10 секунд
6. Добавить логирование запросов

### Данные:
1. Выбрать внешнее API (ЦБ РФ, Yahoo Finance, etc.)
2. Спроектировать структуру ответа JSON
3. Определить формат хранения истории запросов

### Безопасность и надежность:
1. Обработка ошибок внешнего API
2. Валидация данных
3. Ограничение частоты запросов


# Вопросы к команде
### Продакт менеджеру:
1. Как часто пользователи будут запрашивать курс? Это поможет оптимизировать кэширование
2. Нужна ли авторизация для endpoint'а? Или он публичный?
3. Какая точность курса требуется?
4. Нужна ли история больше чем 10 последних запросов? Возможно, нужен отдельный endpoint для расширенной истории
5. Какие метрики успеха для этого функционала? Скорость ответа, доступность, точность?

### Tech Lead'у:
1. Какой внешний API предпочтительнее? Есть ли корпоративные аккаунты?
2. Требования к логированию? Нужно ли логировать все запросы или только ошибки?
3. Какую стратегию кэширования использовать? Redis, memcached, база данных?
4. Нужны ли unit-тесты для этого функционала? Какое покрытие ожидается?
5. Требования к мониторингу? Нужно ли алертить при недоступности внешнего API?
6. Лимиты rate limiting? Только 10 секунд между запросами или дополнительные ограничения?
